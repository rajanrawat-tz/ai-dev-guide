# Contributing to the AI Development Guide

Thank you for your interest in contributing! We welcome improvements to this guide to make it the standard for AI-assisted software development.

## How to Contribute

### 1. Reporting Issues
*   **Bugs/Errors**: If you find a typo, broken link, or technical error, please open an Issue.
*   **Ideas**: If you have an idea for a new section or phase, open a Discussion or Issue to discuss it first.

### 2. Submitting Changes
1.  **Fork** this repository.
2.  Create a new **branch** for your changes: `git checkout -b fix/typo-in-readme` or `git checkout -b feat/new-prompt`.
3.  Make your changes.
    *   Please follow the existing Markdown formatting.
    *   Ensure all links work.
4.  **Commit** your changes with a clear message.
5.  Push to your fork and submit a **Pull Request (PR)**.

## Style Guide

*   **Tone**: Professional, direct, and action-oriented.
*   **Formatting**: Use standard GitHub Flavored Markdown.
*   **Prompts**: When adding prompts, use code blocks and clearly label them.
